Overview and the Basics

The World Wide Web, aka, WWW (yes, that’s what that means) has undergone many changes since it’s inception a few decades ago. If you were indoctrinated into the web with HTML or XHTML, have no fear, your knowledge will not be considered obsolete. If you have only heard rumors of such a thing called HTML, don’t worry, we’ll walk you through the basics. HTML5 is the latest iteration of markup language that makes webpages webpages. So, does that mean if I use HTML4, it wouldn’t work? Well, technically yes. Which is why we still see remnants of the nineties and its, shall we say, distinctly antiquated webpages emerge from the dark every now and then. If it didn’t work, then we couldn’t see it (and suffer).

If you look at the HTML for pages that were created 15 years, 15 months, or even 15 days ago, you will find a wide range of coding specifications running the gamut. (SIDEBAR: to view code of this page, select this) Yet it all works. A related question is why can’t you see the code when you go to a page? The browser you are using renders/translates the code into an attractive (hopefully) combination of images and text. In short, code only works if the browser can understand it. That’s because the code is only as powerful as the browser The HTML tells the browser what to display. How this is displayed (how attractive) is determined by another type of code. Yes, there’s more. So much more. [For more on how to make the what attractive, go on to the CSS section, after this one of course]. Basically, HTML5 is interesting than previous versions, that it allows for all previous iterations to exist. Which is where HTML5 comes in.

As you'll soon see, the language that makes up HTML. Each browser. Not only browsers but browser versions. As browsers update to newer versions to keep up with changing security risks and to offer additional features, there are parts of the HTML language that they stop using or start using. This also means that each version may not support some features of various languages, including HTML.
