There is clearly a lot that went into such a simple looking document, but before we go on, let's define some terminology.

Tags

A tag is the simplest unit of HTML, which basically defines the language. The tags are what the browser reads to determine how to render the code into a webpage. A browser recognizes an HTML tag by the < > which encloses the particular instruction. Like bookends, the < and > lets the browser know when the tag begins (<) and ends (>). Let's take another look at the project code for an example. On the second line, we see the <html> tag. The <html> lets the browser know that the what follows is a webpage.

But wait, there's more. Just as the tag needed to have both an < and an >, to define it as such, the element that the tag modifies needs it as well, which is defined by </>. So think of each tag as a set of bookends, one at the beginning, also called a start tag, < > and one at the end, a closing or end tag < / >. [aside: some browsers allow for a looser format and may not require an ending tag but, best practices require both start and end tags] There are two exceptions to this "bookend" rule: one we discussed earlier - <!DOCTYPE html>which tells the browser which version of html it is using; and the second is </br>, which we'll discuss in the "Structure and Layout" section.

As we return to our example, we can see that the <html> start tag at the beginning, has a closing bookend tag, </html> all the way at the very end of the page. In fact, it's the last tag, which makes sense because it is basically telling the browser that there is nothing more to the webpage.

Elements

An element consists of a set of opening and closing tags and the information it encloses. For example, in the sample project, we see that there are more than one meta elements, which we will learning more about in the next section, each of which begin with the opening meta tag </meta> has some information then end with the closing tag </meta>. The tags by themselves <meta> and </meta>are simply tags.

Sometimes you may see, the term element used interchangeably with tag. Let's take for example the element <!DOCTYPE html>. It is an element that stands by itself. Let's find another example. below that is <html>. but not simply <html>, it also includes the tag and value, lang="en" where the tag is lang and the value is en.
