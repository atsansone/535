Accessing a JavaScript REPL

Most current web browsers ship with a modern Javascript engine and access to it via a developer console. (I think this is the case for MSIE, Firefox, Chrome, and possibly Safari, although some work might be necessary to enable it in all of these.) This section will give a quick tour on how to get up and running in a JavaScript REPL, how they work, etc. My thinking is that if we do this first, the rest of the sections can be done interactively using the console.

The DOM

Eloquent JavaScript doesn't really begin mentioning web programming / the dom specificially until chapter 11/12. I think it should get introducd here earlier. My thinking is, we aren't necessary just trying to introduce JavaScript as a programming language, but rather introduce it within the context of web technologies and the role it plays on the web. So rather than focus on some of the other details Eloquent JavaScript does (object orientation, functional programming, etc.), it would be better to give the reader a basic understanding of programming and then jump into the DOM.

Here again, I struggle with how much detail to go into. Chapter 3 of DOM Scripting might be a good basis though, I think, basically breaking down what the D, O, and M mean.

An Exercise

After an introduction to JavaScript and the DOM, DOM Scripting has a chapter that is basically an exercise in actually putting together a small JS app. I think it would be beneficial to do the same thing here, sort of as an extension of the last section. Also, up until now, I've been assuming that we've just been doing examples in the JS console, and this could be a chance to see how loading scripts on a page works, etc. We could create a template that includes all the HTML or something, and then just have them focus on filling out the js files.
