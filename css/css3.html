CSS3: The Future is Now

The CSS3 specification is still being defined as of 2012, but here are some examples to wet your whistle.

Most browsers already support many of the new CSS3 specifications in their latest versions, and almost all smartphones do. But since not all browsers support these items, it's best to use them to enhance the user's experience on your site. Don't depend on CSS3 elements for the design or construction of your site; those using older browsers may become lost as style rules are applied sporadically or not at all.

Don't assume that people aren't using the latest versions of web browsers out of fear or apathy. Some may want to upgrade and aren't sure how to do it. Some people may be using the "if it ain't broke, don't fix it" philosophy. And some people may be visiting your site from a computer they don't own and cannot upgrade (like a work or lab computer).

Here are a few of the new changes you can make in a browser with CSS3 rules. Note: the display examples will be images rather than CSS3 rules to ensure you will be able to view them, regardless of the browser you're using.

Not sure if browser you're using for this tutorial supports CSS3? You can find out in the "Css3 Support" section of the FindMeByIP website.

RGB Colors

We've been referring to colors in this chapter in hexidecimal (hex) format such as #E01BAB, which is a bright pink color. The hex format identifies the amount of red, green, and blue color to mix into a screen pixel. RGB color does the same thing using a different format: the hex color listed previous would be displayed as rgb(224, 27, 171). The advantage to using RGB colors with CSS3 rules is that you can also set a transparency level for the color, without changing the color value.

Here are the style rules for a class of "box" that can be applied to an HTML5 element:

.box {
  width: 50px;
  height: 50px;
  float:left;
  margin: 10px;
  background-color: rgb(224,27,171);
                     CSS3 rule for color: RGB
}

This is a screenshot of what would be displayed in the browser:

Let's say you wanted to display these boxes as the same color, but with decreasing levels of transparency and ending with the solid color. Before CSS3 rules, you would have to determine what each new hex color would be. Now however, you can determine what your base color would be and apply an alpha transparency between 0 and 1 (with 1 being 100% opaque) with the RGBa format. 

 Create a separate class for each level of transparency
.alpha20 {
  background-color: rgba(224,27,171,0.2);
}
.alpha40 {
  background-color: rgba(224,27,171,0.4);
}
.alpha60 {
  background-color: rgba(224,27,171,0.6);
}
.alpha80 {
  background-color: rgba(224,27,171,0.8);
                     The RGBa property and transparency value
}

Add the new classes to your elements:

<div class="box alpha20"></div>
<div class="box alpha40"></div>
<div class="box alpha60"></div>
<div class="box alpha80"></div>
<div class="box"></div>

And now you have your graduated colors. No static images have to be captured, and they can be easily modified later if you decide to change the color.

Rounded Corners

The Web is...boxy. Your browser is a box and so are all of the elements that are contained inside of it. Before CSS3 you had to created images of rounded corners with a transparent background to create the illusion of an element having rounded corners. The new border-radius declaration in CSS3 allows you to do this very simply. Adding this code to your style rules:

  width: 50px;
  height: 50px;
  float:left;
  margin: 10px;
  background-color: rgb(224,27,171);
  border-radius: 10px;

Vendor Prefixes

Because CSS3 rules aren't completed, many browsers have implemented the rules according to what they think the final specification will look like. A vendor prefix is used when the browser's support for a rule is expected to change once the specification is final. It is a best practice to include all vendor prefixes as well as the generic version of the rule to get the most consistent look across browsers. Here is what the borde-radius rule would look like if all current vendor prefixes were applied in the rule:

.box {

  /* vendor prefixes */
  -moz-border-radius: 10px;  
  -webkit-border-radius: 10px; 
  -o-border-radius: 10px; 
  -ms-border-radius: 10px;

  /* generic rule */
  border-radius: 10px;
}

The vendor prefixes are:

  • -moz- for Mozilla-based (Firefox) browsers
  • -webkit- for Webkit-based (Safari, Chrome) browsers
  • -o- for Opera (Apple's default browser)
  • -ms- for Internet Explorer (Microsoft's default browser)
